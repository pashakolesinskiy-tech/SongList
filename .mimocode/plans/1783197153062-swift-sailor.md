# Fix: alignChords Over-Padding

## Root Cause
Cloned elements inherit `white-space: pre-wrap` from CSS. When chord line has long spaces (e.g. "Dm                              Gm"), `pre-wrap` causes wrapping at container edge. `getBoundingClientRect()` returns container width, not text width → massive over-padding.

## Fix
In `alignChords`, set `white-space: nowrap` on cloned elements before measuring. This forces the browser to compute the full unwrapped text width.

```js
// After cloning:
mChord.style.whiteSpace = 'nowrap';
mLyric.style.whiteSpace = 'nowrap';
```
