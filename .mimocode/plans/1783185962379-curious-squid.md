# Кэш песен для офлайн-доступа

## Проблема
При отсутствии интернета Supabase-запросы падают. localStorage используется как fallback, но данные туда попадают **только** когда Supabase недоступен. Если пользователь работает через Supabase и сеть пропала — localStorage пуст, песни не загружаются.

## Решение: LocalStorage Mirror
Всегда синхронизировать данные с localStorage при каждом успешном��作е Supabase. При офлайне — читать из localStorage.

### Изменения в `js/storage/firebase.js`

**`saveSong`** — после успешного save в Supabase, также сохранять в localStorage:
```
if (supabase success) {
  saveToLocalStorage(record);  // <-- добавить
  return mapFromDb(data);
}
```

**`deleteSong`** — после успешного delete из Supabase, также удалить из localStorage:
```
if (supabase success) {
  deleteFromLocalStorage(id);  // <-- добавить
  return true;
}
```

**`getSongs`** — после успешного fetch из Supabase, обновить localStorage как зеркало:
```
if (supabase success) {
  const songs = data.map(mapFromDb);
  localStorage.setItem('chord-viewer-songs', JSON.stringify(songs));  // <-- добавить
  return songs;
}
```

### Изменения в `sw.js`

Убрать блокировку кэширования Supabase-запросов (строка 55). Это не повлияет на основной функционал, но service worker перестанет пропускать Supabase-запросы мимо кэша:
```
// БЫЛО:
if (url.hostname.includes('supabase') || url.hostname.includes('fonts')) {
  return;
}

// СТАЛО:
if (url.hostname.includes('fonts')) {
  return;
}
```

### Обновить версию кэша
`sw.js`: `CACHE_NAME = 'druisk-v6'` → `'druisk-v7'`

## Файлы для изменения
- `js/storage/firebase.js` — 3 функции (saveSong, deleteSong, getSongs)
- `sw.js` — кэш-политика + версия

## Верификация
1. Открыть приложение, добавить/отредактировать песню → проверить что `localStorage['chord-viewer-songs']` обновляется
2. Отключить сеть (DevTools → Network → Offline) → перезагрузить → песни должны загружаться
3. Удалить песню офлайн → проверить что она удалена из localStorage
