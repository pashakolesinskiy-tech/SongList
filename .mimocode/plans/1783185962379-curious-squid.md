# Исправление поиска: убрать аккорды из результатов

## Проблемы
1. **Аккорды в результатах поиска** — `searchSongs` ищет по `rawText` (строка 128-134 в home.js), который содержит нотации аккордов `[Am]`, `{C}`, `Am7`. Отображаются как есть в `search-excerpt` (строка 183).
2. **Поиск не находит совпадения** — ищет только по `rawText` строкам. Если аккордная строка вида `Am  C  G  D` содержит искомое — покажет её, но не useful. Если строка с текстом содержит искомое — покажет с аккордами.

## Решение

### Файл: `js/views/home.js`

**1. Добавить функцию `stripChords(text)`** (перед `searchSongs`):
```js
function stripChords(text) {
  return text
    .replace(/\{[^}]*\}/g, '')   // ChordPro: {Am7}
    .replace(/\[[^\]]*\]/g, '')   // Inline: [Am]
    .replace(/^([A-H][#b]?(?:maj|min|dim|aug|sus[24]?|add\d+)?m?(?:\/[A-H][#b]?)?(?:\d+)?\s*)+$/gm, ''); // Chord-only lines
}
```

**2. Изменить `searchSongs`** — искать по очищенному тексту:
- `rawText` → `stripChords(rawText)` для поиска совпадений
- `matchingLines` собирать из очищенного текста

**3. Изменить `renderSearchResults`** — отображать очищенные строки:
- Передать в `highlightMatch` уже очищенную строку

## Верификация
1. Ввести в поиск слово из текста песни → в результатах показываются только строки с текстом, без аккордов
2. Ввести аккорд (например "Am") → он НЕ должен отображаться в результатах как текст
3. Проверить что поиск по названию и исполнителю работает как раньше
