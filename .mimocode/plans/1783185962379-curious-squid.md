# Поддержка аккордов в немецкой нотации (H, H7, Hm и т.д.)

## Проблема
Все регулярные выражения для распознавания аккордов используют `[A-G]` — не включают `H` (немецкая нотация для B-натураль). Аккорды вроде H7, Hm, Hdim не распознаются и не транспонируются.

## Затронутые файлы (8 мест)

### 1. `js/parser/chordpro.js:1` — CHORD_REGEX
`[A-G]` → `[A-H]`

### 2. `js/parser/plain-text.js` — 5 регулярных выражений
- Строка 1: `INLINE_CHORD_REGEX` — `[A-G]` → `[A-H]`
- Строка 2: `CHORD_LINE_REGEX` — `[A-G]` → `[A-H]`
- Строка 3: `CHORD_TOKEN_REGEX` — `[A-G]` → `[A-H]`
- Строка 7: `detectFormat` chordpro pattern — `[A-G]` → `[A-H]`
- Строка 8: `detectFormat` inline pattern — `[A-G]` → `[A-H]`

### 3. `js/views/song.js:87` — extractChords
`[A-G]` → `[A-H]`

### 4. `js/utils/transpose.js` — транспозиция
- Строка 18: `transposeChord` regex — `[A-G]` → `[A-H]`
- Строка 42: `transposeSongText` regex — `[A-G]` → `[A-H]`
- Строки 1-2: Добавить `H` в SHARP_NOTES и FLAT_NOTES массивы:
  ```
  SHARP_NOTES = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B','H']
  FLAT_NOTES  = ['C','Db','D','Eb','E','F','Gb','G','Ab','A','Bb','B','H']
  ```
  H маппится на тот же семитон что и B (индекс 11), просто как альтернативное имя.
- Функция `noteToSemitone`: добавить проверку для `H` → вернуть 11 (тот же индекс что B)

## Верификация
1. Ввести текст с аккордом `H7` → он должен распознаться и отобразиться как аккорд
2. Транспозиция аккорда `H` на +1 → `C`, на -1 → `A`
3. Аккорды `Hm`, `Hdim`, `Haug` — всё должно работать
