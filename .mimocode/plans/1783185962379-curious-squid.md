# Исправление клика по сердечку

## Проблема
Кнопка сердечка находится внутри `<a href="#/song/...">`. `stopPropagation()` не блокирует навигацию — браузер обрабатывает `<a>` на уровне выше. Клик по сердечку открывает песню. Также сердечко "пустеет" при клике на другие объекты из-за делегирования событий.

## Решение: вынести кнопки из `<a>`

### `js/views/home.js` — изменить структуру HTML в `renderList`

**БЫЛО:**
```html
<a href="#/song/${song.id}">
  <div class="song-card">
    ...text...
    <div class="song-actions">
      <button class="btn-fav">♥</button>
      <button class="btn-delete">✕</button>
    </div>
  </div>
</a>
```

**СТАЛО:**
```html
<div class="song-card">
  <a href="#/song/${song.id}" class="song-link">
    ...text...
  </a>
  <div class="song-actions">
    <button class="btn-fav">♥</button>
    <button class="btn-delete">✕</button>
  </div>
</div>
```

Кнопки теперь вне `<a>` — клик по ним не триггерит навигацию.

### `css/style.css` — обновить `.song-card`

Сделать `.song-card` flex-контейнером, а `.song-link` — `flex:1` с `overflow:hidden`:
```css
.song-card {
  display: flex;
  align-items: center;
}
.song-link {
  flex: 1;
  overflow: hidden;
}
```

## Файлы
- `js/views/home.js` — HTML-структура в `renderList`
- `css/style.css` — `.song-card` и `.song-link`

## Верификация
1. Клик по сердечку — только переключает избранное, НЕ открывает песню
2. Клик по тексту песни — открывает песню как раньше
3. Клик по удалению — работает как раньше
