# Задача: Настройка тональности + Исправление скролла настроек

## Задача 1: Изменение начальной тональности песни

### Что нужно
Пользователь с кодом разблокировки должен иметь возможность менять исходную тональность песни. Сейчас транспозиция работает как смещение от исходных аккордов в `rawText`. Нужно добавить поле `originalKey` в песню и UI для его изменения.

### Файлы для изменения

1. **`js/app.js`** — в `updateSongActions()` (строки 386–475):
   - Добавить select с вариантами тональностей (C, C#/Db, D, D#/Eb, E, F, F#/Gb, G, G#/Ab, A, A#/Bb, B)
   - При изменении — сохранять `originalKey` в песню через `saveSong()`
   - Показывать текущую тональность рядом с кнопками транспозиции

2. **`js/views/song.js`** — отображать `originalKey` в заголовке (строки 54–65):
   - Показать "Тональность: X" если `originalKey` задан

### Логика
- Поле `originalKey` хранится в объекте песни (Supabase + localStorage)
- При отображении: если `originalKey` задан, показывать его вместо автоопределения через `detectSongKey()`
- Кнопки +/- для транспозиции работают относительно `originalKey`
- Сброс транспозиции возвращает к `originalKey`

## Задача 2: Исправление скролла settings sheet

### Проблема
`.settings-sheet` не имеет `overflow` и `max-height`, поэтому при большом количестве настроек (особенно когда разблокировано — добавляются Supabase URL/Key) контент обрезается и не скроллится.

### Файл для изменения

**`css/style.css`** — строки 476–494, класс `.settings-sheet`

### Изменение
Добавить в `.settings-sheet`:
```css
max-height: 85vh;
overflow-y: auto;
```

## Верификация

1. Открыть приложение, разблокироваться кодом `080826`
2. Открыть настройки — убедиться что контент скроллится если не влезает
3. Открыть песню → ⚙️ → проверить что появился выбор тональности
4. Изменить тональность → сохранить → перезагрузить → проверить что тональность сохранилась
5. Проверить что транспозиция +/- работает относительно выбранной тональности
