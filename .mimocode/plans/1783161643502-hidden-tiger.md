## Разбиение `app.js` (538 → 3 файла)

### Новые файлы

1. **`js/settings.js`** (~120 строк) — конфигурация и настройки
   - `config` объект + загрузка из localStorage
   - `settings` объект + загрузка из localStorage
   - `saveSettings()`, `applySettings()`
   - Экспорт: `config`, `settings`, `saveSettings`, `applySettings`

2. **`js/lock.js`** (~100 строк) — система блокировки
   - `LOCK_CODE`
   - `unlocked` состояние
   - `requestUnlock()`, `showLockModal()`, `lockApp()`
   - Экспорт: `unlocked`, `requestUnlock`, `lockApp`

3. **`js/song-actions.js`** (~120 строк) — действия с песней в settings sheet
   - `detectSongKey()`
   - `updateSongActions()`
   - Экспорт: `updateSongActions`

4. **`js/app.js`** (~100 строк) — только роутинг и shell
   - Импорты из новых модулей
   - `renderShell()` (только HTML + навешивание обработчиков через импорты)
   - `navigate()`, `parseHash()`, `updateHeader()`
   - `showConfirm()` (модальное окно)

### Структура импортов

```
app.js ──┬── settings.js (config, settings, saveSettings, applySettings)
         ├── lock.js (unlocked, requestUnlock, lockApp)
         ├── song-actions.js (updateSongActions)
         ├── views/*.js
         └── parser/*.js
```

## CSS (749 строк) — оставить как есть

CSS обычно держат в одном файле. Разбиение на модули требует сборщика (PostCSS, CSS Modules), что не оправдано для этого проекта.

## Порядок выполнения

1. Создать `js/settings.js` — вынести конфиг и настройки
2. Создать `js/lock.js` — вынести систему блокировки
3. Создать `js/song-actions.js` — вынести detectSongKey и updateSongActions
4. Обновить `js/app.js` — оставить только роутинг и shell, импортировать новое
5. Создать `js/views/favorites.js` — вынести логику избранного
6. Обновить `js/views/home.js` — импортировать favorites
